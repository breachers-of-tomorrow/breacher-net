# Security Policy

## Reporting a Vulnerability

If you discover a security vulnerability in breacher.net, **please report it responsibly**.

### How to Report

1. **GitHub Security Advisories (preferred):** Use [GitHub's private vulnerability reporting](https://github.com/breachers-of-tomorrow/breacher-net/security/advisories/new) to submit a confidential report.
2. **Discord DM:** Contact a Keeper of The Manifest via the [Breachers of Tomorrow Discord](https://discord.gg/sGeg5Gx2yM).

**Do not open a public issue for security vulnerabilities.**

### What to Include

- Description of the vulnerability
- Steps to reproduce
- Potential impact
- Suggested fix (if you have one)

### Response

- We aim to acknowledge reports within **48 hours**
- We'll work with you to understand and resolve the issue
- Credit will be given in the fix unless you prefer to remain anonymous

## Scope

This policy covers:

- The breacher.net web application (`src/`)
- API routes (`src/app/api/`)
- Python poller scripts (`poller/`)
- Docker and deployment configurations

This policy does **not** cover:

- cryoarchive.systems (that's Bungie's site — report to them directly)
- Third-party dependencies (report upstream, but let us know too)

## Supported Versions

| Version | Supported |
|---------|-----------|
| Latest on `main` | Yes |
| Older releases | Best effort |
